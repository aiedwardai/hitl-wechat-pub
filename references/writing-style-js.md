# 剑识 Writing Style (writing-style-js)

基于 16Miku/wechat-auto-publishing 原版 writing-style.md 改编，融合剑识（剑识此刻公众号）的实际写作经验。

## Goal

写出有态度、有判断力的 AI/科技评论文章。读者看完能带走一个观点，不是一堆新闻。

## Style principles

- **判断在前，数据在后**：每段先给观点，再用数据支撑，不要反过来
- **第一人称 + 直接对读者讲话**：用"我判断"、"说实话"、"你注意"、"我告诉你"。不要"我们认为"、"作者认为"
- **语感锐利**：短句为主，一针见血。不要圆滑，不要"在某种程度上"
- **情绪但克制**：有态度但不上头，冷静的狠劲
- **结尾收成判断**：不要让读者自己琢磨"所以呢"，直接给结论

## Title guidance

格式：`剑识 | [单刀直入的核心观点，20字以内]`

不要"剑识此刻"，只保留"剑识"。标题就是文章的核心判断，不做铺垫。

示例：
- ✅ `剑识 | AI芯片的超级周期还早`
- ✅ `剑识 | Fable 5离线暴露的问题比宕机更严重`
- ❌ `剑识此刻 | AI芯片分析`
- ❌ `剑识 | 今日AI新闻汇总`

## Opening guidance

**第一句就抛核心判断/反常识观点，不做铺垫。**

不要"今天我们来聊聊"、"最近AI圈很热闹"。读者只有 2 秒决定要不要看，开头就要抓住他。

示例：
- ✅ "我告诉你一件事：AI 芯片超级周期才刚开始，大部分人还没反应过来。"
- ✅ "Fable 5 还黑着。第4天了。比宕机更让人担心的是——没人知道什么时候好。"
- ❌ "最近AI行业发生了一些重大事件，Fable 5 已经离线四天……"

## Body structure

正文不设固定板块顺序。按逻辑推进，每个段落独立成节。

```markdown
1. opening — 第一句抛结论
2. section — 一个角度/一个论点
3. section — 另一个角度/另一个论点
4. section — 第三个角度/另一个维度
5. 剑识 — 综合判断 + 前瞻观点
```

### 标题格式

**不使用数字编号**（不用 `## 1、`、`## 2、`），直接用纯文字标题：

```markdown
## 第一段的主题
正文2-4句。

## 第二段的主题
正文2-4句。

## 剑识
综合判断收束。
```

### 每段写法

短段，2-4 句。先给判断，再给数据支撑。不要反过来第一句念数据第二句才给观点。

✅ 正确写法：
```markdown
我判断：Marvell 踩中的不是芯片周期，而是基础设施重构周期。
以前芯片涨跟着 PC 换机、手机出货走，现在跟着 AI 数据中心对互联的需求走。
```

❌ 错误写法：
```markdown
Marvell 年内涨了300%，6月22日将迎来关键催化剂事件，市场预期其将进入新指数。
这说明芯片板块正在走强。
```

### 剑识段（收束段）

不叫"我的判断"，不叫"结论"，叫**剑识**。

格式：
```markdown
## 剑识

[2-4 句综合判断，直接给观点]

接下来盯着[核心变量]——因为[为什么这个变量决定方向]。
```

示例：
```markdown
## 剑识

三个理由告诉我，这轮超级周期还在上半场。

接下来盯着三件事：Marvell 催化剂落地后的机构资金跟进速度，
SNDK 换股完成后的估值重定价，以及 NVIDIA Rubin 路线图是否继续加速——
因为迭代越快，竞争对手的时间窗口就越窄。
```

## Publishable Markdown conventions

使用 explicit frontmatter，注意**不要在标题中写"剑识此刻"**：

```markdown
---
title: 剑识 | AI芯片超级周期还早
author: 戴剑
summary: 一句话摘要
cover: ./cover.png
---
```

## Suggested publish-ready pattern

```markdown
---
title: 剑识 | [核心观点 ≤20字]
author: 戴剑
summary: [一句话摘要]
cover: ./cover.png
---

第一句直接抛判断/反常识观点。

第二句展开核心逻辑。

## [第一个论点]

判断在前，数据在后。2-4句。

## [第二个论点]

另一个角度的判断+数据。

## [第三个论点]

第三个维度的判断+数据。

## 剑识

综合判断，直接给观点。

接下来盯着[核心变量]——因为[为什么这个变量决定方向]。
```

## Writing quality rule

来源于原版 writing-style.md 的实战经验：

**不要把搜索到的新闻标题直接粘贴到正文。**

### Avoid

- raw headline insertion（直接粘贴新闻标题）
- truncated fragments ending with `...`（截断片段）
- media-report phrasing such as `报道：` left untouched（媒体前缀）
- title-like syntax being treated as final paragraph content

### Prefer

1. 清理来源标题
2. 去掉媒体式前缀（"报道："、"据悉"、"据了解"）
3. 理解事件含义
4. 用目标账号的语音重新写
5. 确保每段读起来像完整段落的叙述，不是素材占位符

### Example

**Weak body sentence:**
```markdown
"48小时通牒"与"45天停火协议"挑动市场，周一油价冲高回落，黄金下跌、美股期货转涨，...
```

**Better rewritten paragraph:**
```markdown
停火预期一出来，市场情绪立刻被带着跑，油价先冲高，随后又回落，黄金和风险资产也跟着来回波动。真正难的不是没有方向，而是消息反复，节奏特别容易做错。
```

## Avoid

- ❌ 分步骤教学/教程
- ❌ 写代码块
- ❌ 工具推荐/横向对比
- ❌ 中立圆滑的表述（"在某种程度上"、"从某种意义上说"）
- ❌ 固定 Slogan/坐右铭
- ❌ 点赞转发等 CTA（剑识不需要）
- ❌ "作者提示：个人观点，仅供参考"

## 原版 writing-style.md 与本版差异

| 维度 | 原版 writing-style.md | 剑识 (writing-style-js) |
|------|----------------------|-------------------------|
| 标题 | 通用 | `剑识 \| [观点 ≤20字]` |
| 编号 | `## 1、` 数字编号 | 无编号，纯文字标题 |
| 收束段 | `我的操作？` / `明日观察重点` | `剑识` — 综合判断 |
| 结尾 | 互动句 + CTA | 观点句自然收尾 |
| 人称 | 无要求 | 第一人称（我判断、说实话） |
| 结构 | 5段固定模板 | 灵活，按论点推进 |
